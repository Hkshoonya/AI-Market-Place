## Tech Debt Summary

### LOW Priority (no blockers)

1. **Watchlist routes missing handleApiError** (Phase 9/OBS-01) — 3 API routes in `src/app/api/watchlists/` don't capture errors to Sentry. Errors fall through to default Next.js 500 handler. **Fix:** Add `handleApiError` wrapper to the 3 route files.

2. **SWR_TIERS underutilized** (Phase 14/PERF-01) — `SWR_TIERS` constants (FAST/MEDIUM/SLOW) exported from `config.ts` but only ~4 of 44 SWR consumers import them. Most components set `refreshInterval` ad-hoc or omit polling. Not a functional break — SWR global config still provides deduping and revalidation. **Fix:** Migrate per-component polling literals to `SWR_TIERS`.

3. **Auth E2E Test 1 soft failure** (Phase 15/E2E-02) — Pre-existing: error alert element renders but may have empty text when Supabase client throws ENOTFOUND in offline mode. Documented in 18-01-SUMMARY. Not introduced by any v1.1 phase. **Fix:** Make assertion check `role="alert"` visibility without requiring specific text content.

4. **SUMMARY frontmatter gaps** (Phase 15) — Plans 15-01 and 15-03 lack `requirements-completed` in frontmatter. E2E-01, E2E-02, E2E-05, E2E-06 confirmed satisfied by VERIFICATION and REQUIREMENTS cross-reference. Documentation-only gap.

**Total: 4 low-priority items. Zero blockers. Zero integration breaks.**
