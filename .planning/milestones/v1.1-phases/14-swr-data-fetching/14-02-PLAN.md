<!-- Conversion pattern (from research) -->
```typescript
// BEFORE:
const [data, setData] = useState<T | null>(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState<string | null>(null);
useEffect(() => {
  fetch(url).then(r => r.json()).then(d => setData(d))
    .catch(() => setError("Failed")).finally(() => setLoading(false));
}, []);

// AFTER:
const { data, error, isLoading } = useSWR<T>(url, { ...SWR_TIERS.TIER });
```

  <action>
For each component, apply the standard conversion pattern:

1. Add `import useSWR from 'swr'` and `import { SWR_TIERS } from '@/lib/swr/config'`.
2. Remove the `useState` calls for `data`, `loading`/`setLoading`, and `error`/`setError`.
3. Remove the `useEffect` that calls `fetch()`.
4. Replace with `const { data, error, isLoading } = useSWR<DataType>(url, { ...SWR_TIERS.TIER })`.
5. Update the component body to use `isLoading` instead of `loading`, and `error` (which is now an Error object, not a string) instead of the old error state.
6. For error display, use `error?.message` or a generic fallback string.

**Specific files and tiers:**

a. `market-ticker.tsx` -- FAST tier
   - Replace: `useState<TickerItem[]>([])` + `useEffect fetch('/api/charts/ticker')`
   - With: `const { data: items } = useSWR<TickerItem[]>('/api/charts/ticker', { ...SWR_TIERS.FAST })`
   - Use `items ?? []` for the empty fallback. Keep `if ((items ?? []).length === 0) return null`.

b. `top-movers.tsx` -- FAST tier
   - Replace: 3 useState + useEffect
   - With: `const { data, error, isLoading } = useSWR<TopMoversData>('/api/charts/top-movers?limit=10', { ...SWR_TIERS.FAST })`
   - Keep `tab` useState (UI state, not data fetching).

c. `trading-chart.tsx` -- MEDIUM tier
   - Has dynamic params (metric, range, model slug). Build URL from params: `` `/api/charts/trading?${params.toString()}` ``
   - SWR auto-refetches when the key (URL) changes, so remove any manual refetch logic.
   - Keep all UI state (selected metric, range, etc.) as useState.

d. `rank-timeline.tsx` -- MEDIUM tier
   - Has dynamic params (slugs, days, metric). Build URL: `` `/api/charts/rank-timeline?${params.toString()}` ``
   - SWR key changes trigger automatic refetch. Remove useCallback+useEffect fetch pattern.

e. `quality-price-frontier.tsx` -- MEDIUM tier
   - Has filter params. Build URL with query string. SWR handles refetch on param change.

f. `benchmark-heatmap.tsx` -- MEDIUM tier
   - Has filter params. Same pattern as quality-price-frontier.

g. `listing-reviews.tsx` -- SLOW tier (reviews rarely change, but has mutation)
   - Replace fetch with useSWR. For the review submission (POST), keep the fetch POST but call `mutate()` after successful submission to revalidate the list.
   - Destructure `{ data, error, isLoading, mutate }` from useSWR.

h. `english-bid-panel.tsx` -- FAST tier (auction bids are time-sensitive)
   - Replace fetch with useSWR. For bid submission (POST), keep the POST fetch and call `mutate()` after.

For components with mutations (listing-reviews, english-bid-panel):
- Keep the existing `fetch(url, { method: 'POST', ... })` for write operations.
- After a successful POST, call `mutate()` from the useSWR hook to revalidate the data.
- Do NOT use `useSWRMutation` -- the simple `mutate()` pattern is sufficient and consistent.
