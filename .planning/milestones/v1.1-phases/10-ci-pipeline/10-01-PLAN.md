  <name>Task 1: Create CI workflow with parallel lint, typecheck, and test jobs</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create `.github/workflows/ci.yml` with the following structure:

**Workflow name:** `CI`

**Triggers:**
- `pull_request` with `branches: [main]` (fires only for PRs targeting main)
- `workflow_dispatch` (manual trigger for testing)
- NO push triggers (conserve Actions minutes, per user decision)

**Concurrency:** Add concurrency group to cancel in-progress runs on new pushes to same PR:
```yaml
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
```

**Three parallel jobs, each with identical setup steps:**
1. `lint` (name: "Lint") -- runs `npm run lint`
2. `typecheck` (name: "Typecheck") -- runs `npx tsc --noEmit`
3. `test` (name: "Test") -- runs `npm test`

Each job follows this pattern:
- `runs-on: ubuntu-latest`
- `timeout-minutes: 5`
- Steps: `actions/checkout@v4`, `actions/setup-node@v4` with `node-version: 22` and `cache: 'npm'`, `npm ci`, then the specific check command

**Important details:**
- Job keys are lowercase (`lint`, `typecheck`, `test`) -- these determine status check names shown on PRs as "CI / Lint", "CI / Typecheck", "CI / Test"
- Use `name:` field on each job for readable display names (Lint, Typecheck, Test)
- No env vars needed -- ESLint config has no type-aware linting (no `parserOptions.project`), all 170+ tests are pure unit tests with no external dependencies
- No coverage upload -- deferred to Phase 12
- No .next/cache caching -- tsc --noEmit doesn't use it

After creating the file, verify the three commands work locally:
- `npm run lint`
- `npx tsc --noEmit`
- `npm test`

  <action>
Human must configure branch protection in the GitHub UI. This cannot be automated without an admin-scoped token. The CI workflow must have run at least once before GitHub shows the check names in the branch protection dropdown (Pitfall 4 from research).

**Step 1: Trigger the workflow**
1. Push the CI workflow file to a branch (or it will run on the next PR to main)
2. Open a PR targeting main -- this triggers the CI workflow
3. Wait for all 3 checks to complete (should take ~2 minutes)

**Step 2: Configure branch protection**
1. Go to https://github.com/Hkshoonya/AI-Market-Place/settings/branches
2. Click "Add branch protection rule" (or edit existing rule for main)
3. Branch name pattern: `main`
4. CHECK: "Require status checks to pass before merging"
5. CHECK: "Require branches to be up to date before merging"
6. In the search box, search and add these 3 status checks:
   - `CI / Lint`
   - `CI / Typecheck`
   - `CI / Test`
7. LEAVE UNCHECKED: "Require pull request reviews before merging" (solo developer, per user decision)
8. Do NOT add a CODEOWNERS file
9. Click "Create" or "Save changes"

**Step 3: Verify enforcement**
1. Create a test branch, introduce a type error (e.g., `const x: number = "string"` in any .ts file)
2. Push and open a PR to main
3. Verify the Typecheck job fails (red X)
4. Verify the merge button is disabled or shows "merging is blocked"
5. Close the test PR and delete the test branch
