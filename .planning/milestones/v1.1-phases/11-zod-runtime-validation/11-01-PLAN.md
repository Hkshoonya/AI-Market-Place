  <name>Task 1: Create parseQueryResult utilities with tests</name>
  <files>src/lib/schemas/parse.ts, src/lib/schemas/parse.test.ts</files>
  <behavior>
    - parseQueryResult with valid response and valid data returns validated T[]
    - parseQueryResult with Supabase error in response returns empty array
    - parseQueryResult with null data returns empty array
    - parseQueryResult with data that fails schema validation returns empty array
    - parseQueryResult calls Sentry.captureException on validation failure with tags { "error.type": "schema_validation" }
    - parseQueryResult Sentry call includes fingerprint ["schema-validation", schemaName]
    - parseQueryResult Sentry extras include issues array but NOT raw data
    - parseQueryResultSingle with valid response returns validated T
    - parseQueryResultSingle with Supabase error returns null
    - parseQueryResultSingle with data that fails schema validation returns null and calls Sentry
    - reportSchemaError wraps Sentry call in try-catch so client-side (where Sentry may not be initialized) falls back to console.error
  </behavior>
  <action>
    1. Create `src/lib/schemas/parse.test.ts` first with all tests above. Mock `@sentry/nextjs` using `vi.mock("@sentry/nextjs")`. Use a simple test schema like `z.object({ id: z.string(), name: z.string() })` for test data.

    2. Run tests -- they should all fail (RED).

    3. Create `src/lib/schemas/parse.ts` implementing:
       - `SupabaseResponse` type: `{ data: unknown; error: { message: string; code?: string } | null }`
       - `parseQueryResult<T>(response, schema, schemaName)`: checks error first, then validates `z.array(schema).safeParse(response.data)`, returns `T[]` or `[]`
       - `parseQueryResultSingle<T>(response, schema, schemaName)`: checks error first, then validates `schema.safeParse(response.data)`, returns `T | null`
       - `reportSchemaError(schemaName, error)`: private function that calls `Sentry.captureException` with `tags: { "error.type": "schema_validation" }`, `fingerprint: ["schema-validation", schemaName]`, `extra: { schemaName, issueCount, issues: error.issues.map(...) }`. Wrapped in try-catch with `console.error` fallback. Does NOT include raw data per locked decision.
       - Use `.safeParse()` everywhere per locked decision -- never `.parse()`.

    4. Run tests -- they should all pass (GREEN).
  </action>
  <verify>
    <automated>npx vitest run src/lib/schemas/parse.test.ts -x</automated>
  </verify>
  <done>parseQueryResult and parseQueryResultSingle utilities exist with full test coverage. Sentry reporting uses correct tags, fingerprint, and extras. Try-catch fallback handles client-side Sentry absence.</done>
</task>
