    - `replicate.ts`: Export `KnownReplicateModel` interface (owner, name, description, category, run_count, is_open_weights) and `REPLICATE_KNOWN_MODELS: KnownReplicateModel[]`. Keep as array — do NOT convert to Record. Copy all model entries from `replicate.ts`.

    CRITICAL: Copy data exactly as-is from each adapter. Zero behavior change. Do not reformat or reorder fields.

    **Step 3 — Create `src/lib/data-sources/shared/infer-category.ts`:**

    Unify all 5 `inferCategory()` implementations into a single function with mode dispatch:

    - Define `InferCategoryMode = "id" | "description" | "arch" | "topics"`
    - Define `InferCategoryOptions` with mode + optional fields (modelId, description, arch, topics)
    - `inferCategory(opts)` uses switch on `opts.mode`:
      - `"id"`: Merge OpenAI + Google prefix rules into `ID_PREFIX_CATEGORY` array. Use `.startsWith()` matching. Default: "specialized"
      - `"description"`: Port Replicate's keyword matching into `DESC_KEYWORD_CATEGORY`. Default: "specialized"
      - `"arch"`: Port OpenRouter's modality set logic (output_modalities check for image/video/audio, then input multimodal check). Default: "llm"
      - `"topics"`: Port GitHub trending's combined topics+description matching. Default: "specialized"
    - `inferModalities(modelId)`: Merge OpenAI + Google implementations. Prefix-based matching for vision/image/audio/video/embedding model IDs. Default: `["text"]`

    CRITICAL: mode parameter must be required, not defaulted. Keyword maps inline in the module (per user decision). inferCategory module must NOT import from build-record (accepts only primitives — avoids circular dependency per Pitfall 5 in RESEARCH.md).
