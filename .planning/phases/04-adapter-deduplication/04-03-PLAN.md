  <files>src/lib/data-sources/shared/adapter-syncer.ts</files>
  <action>
    Create `src/lib/data-sources/shared/adapter-syncer.ts` implementing the factory per RESEARCH.md "Pattern 4".

    **Define `AdapterSyncerConfig<TApiResult>` interface:**
    - `apiKeySecret: string` — env var name (e.g., "ANTHROPIC_API_KEY")
    - `apiSourceName: string` — human-readable name for metadata (e.g., "anthropic_api")
    - `knownModelIds: string[]` — all static model IDs
    - `buildRecordFn: (modelId: string, overrides?: Partial<KnownModelMeta>) => Record<string, unknown>` — bound buildRecord call
    - `staticModelCount: number` — for healthCheck message
    - `scrapeFn: (signal?: AbortSignal) => Promise<string[]>` — scrape attempt, returns model IDs found or []
    - `apiFn: (apiKey: string, signal?: AbortSignal) => Promise<TApiResult | null>` — API fetch, returns null on failure
    - `enrichFn: (recordMap: Map<string, Record<string, unknown>>, apiResult: TApiResult, now: string, buildRecordFn: ...) => void` — enrich map with API results
    - `healthCheckUrl: string`
    - `healthCheckHeaders: (apiKey: string) => Record<string, string>`
    - `healthCheckSuccessMsg: string`

    **Implement `createAdapterSyncer<TApiResult>(config)` returning `{ sync, healthCheck }`:**

    `sync(context: SyncContext): Promise<SyncResult>`:
    1. Build recordMap: `new Map()` populated by iterating `config.knownModelIds` and calling `config.buildRecordFn(modelId)` for each. Set metadata sources = ["static_data"]
    2. Try scrape: call `config.scrapeFn(context.signal)`. For each scraped model ID not already in recordMap, add via `config.buildRecordFn(modelId)`. If scrape found models, push scrape source name to metadata sources.
    3. Check API key: `process.env[config.apiKeySecret]`. If present, call `config.apiFn(apiKey, context.signal)`. If result, call `config.enrichFn(recordMap, result, now, config.buildRecordFn)`. Push config.apiSourceName to metadata sources.
    4. Upsert: call `upsertBatch(context, Array.from(recordMap.values()))` from utils.ts
    5. Return SyncResult with model count, sources, metadata

    `healthCheck(): Promise<HealthCheckResult>`:
    1. Check API key from env. If missing, return healthy with static-only message including `config.staticModelCount`.
    2. If key present, use `fetchWithRetry` to ping `config.healthCheckUrl` with `config.healthCheckHeaders(apiKey)`.
    3. Return healthy/unhealthy based on response.

    Import types from `../types` (SyncContext, SyncResult, HealthCheckResult). Import `upsertBatch`, `fetchWithRetry` from `../utils`. Import `KnownModelMeta` from `./build-record`.

    CRITICAL: Read each of the 3 existing adapter sync() implementations carefully to ensure the factory captures the exact pipeline structure. Pay special attention to:
    - How scraped model IDs are used (they add to recordMap if not already present)
    - The order: static first, scrape second, API third
    - upsertBatch call with the full records array
  </action>
