<task type="auto">
  <name>Task 1: Create railway.json configuration</name>
  <files>railway.json</files>
  <action>
    Create `railway.json` in project root with Railway service configuration:

    ```json
    {
      "$schema": "https://railway.com/railway.schema.json",
      "build": {
        "builder": "DOCKERFILE",
        "dockerfilePath": "Dockerfile"
      },
      "deploy": {
        "startCommand": "node server/custom-server.js",
        "healthcheckPath": "/api/health",
        "healthcheckTimeout": 30,
        "restartPolicyType": "ON_FAILURE",
        "restartPolicyMaxRetries": 5
      }
    }
    ```

    Notes:
    - The `startCommand` overrides the Dockerfile CMD when present — this ensures Railway uses our custom server even if the Dockerfile CMD is cached. Both point to the same file so it's consistent either way.
    - `healthcheckPath` tells Railway to probe /api/health — if it returns non-200 for healthcheckTimeout seconds, the deploy is rolled back.
    - `restartPolicyType: ON_FAILURE` with max 5 retries prevents infinite crash loops while allowing recovery from transient failures.
  </action>
  <verify>
    <automated>node -e "const c = require('./railway.json'); console.assert(c.deploy.healthcheckPath === '/api/health'); console.assert(c.build.builder === 'DOCKERFILE'); console.log('railway.json valid');"</automated>
  </verify>
  <done>
    - railway.json exists with Dockerfile builder, healthcheck path, and restart policy
    - JSON is valid and parseable
  </done>
</task>

  <what-built>Railway deployment is running. Now DNS needs to point aimarketcap.tech to Railway.</what-built>
  <action>
    The user must configure Cloudflare DNS and Railway custom domain. Claude cannot access these dashboards.
  </action>
  <how-to-verify>
    **Step 1: Add custom domain in Railway**
    1. Railway Dashboard -> Service -> Settings -> Networking
    2. Click "Custom Domain" -> enter `aimarketcap.tech`
    3. Railway will show you a CNAME target (e.g., `<something>.up.railway.app`)
    4. Also add `www.aimarketcap.tech` as a second custom domain

    **Step 2: Configure Cloudflare DNS**
    1. Cloudflare Dashboard -> aimarketcap.tech -> DNS -> Records
    2. Add CNAME record:
       - Type: `CNAME`
       - Name: `@` (root)
       - Target: the Railway CNAME target from Step 1
       - Proxy status: **DNS only** (grey cloud) — Railway handles SSL
    3. Add CNAME record for www:
       - Type: `CNAME`
       - Name: `www`
       - Target: the Railway CNAME target from Step 1
       - Proxy status: **DNS only** (grey cloud)
    4. If Cloudflare has an existing A record for `@`, delete it first
    5. Ensure Cloudflare SSL/TLS mode is set to "Full" (Settings -> SSL/TLS)

    **Step 3: Wait for DNS propagation** (usually 1-5 minutes with Cloudflare)

    **Step 4: Verify**
    1. `curl -I https://aimarketcap.tech` -> should return 200 with valid SSL
    2. `curl -I https://www.aimarketcap.tech` -> should return 301 redirect to `https://aimarketcap.tech`
    3. Visit https://aimarketcap.tech in browser -> app loads correctly
    4. Visit https://aimarketcap.tech/api/health -> returns JSON with status

    **Step 5: Verify cron jobs are running**
    Check Railway logs after ~6 hours (or manually trigger from admin dashboard):
    - Look for `[cron] Starting: Tier 1 Sync` in logs
    - OR visit /admin/data-sources and click "Sync Now" on any adapter to verify the sync pipeline works in production
