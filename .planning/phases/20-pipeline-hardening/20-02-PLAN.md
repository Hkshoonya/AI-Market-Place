<objective>
Make sync failures visible: the cron route returns per-adapter error details in its response body, the orchestrator fires Sentry alerts on repeated failures and logs structured entries, and existing retry + healthCheck capabilities are verified by tests.

Purpose: Currently the cron sync route returns a terse summary with `errors: d.errors.length` (count only, no messages). Adapter failures are invisible -- no Sentry alert, no structured log entry with failure context. This plan fixes failure visibility from orchestrator (Sentry + system_log) through the cron response body (error messages exposed).

Output: Updated cron route with error messages in body, orchestrator with Sentry alerting + structured failure logging, and verification tests for existing retry and healthCheck behavior.
</objective>
