<objective>
Seed the data_sources table programmatically on startup, validate secrets with fail-fast for core secrets, and refactor resolveSecrets() to return structured results.

Purpose: The data_sources table is likely unseeded in Supabase (no manual SQL on first deploy). Core secrets being silently absent causes the pipeline to run but fail mysteriously. This plan fixes both by making startup explicit and loud about missing configuration.

Output: Three new modules (seed-config.ts, seeder.ts, startup.ts), refactored resolveSecrets(), and instrumentation.ts wiring.
</objective>
