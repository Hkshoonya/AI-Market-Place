<objective>
Convert 13 components that use direct Supabase client queries (createClient() + supabase.from() in useEffect or useCallback) to useSWR hooks with inline fetcher functions and 'supabase:' prefixed cache keys.

Purpose: These components bypass API routes and query Supabase directly. They can't use the global jsonFetcher (which expects a URL). Instead, each gets an inline async fetcher that performs the Supabase query. The SWR cache key uses a 'supabase:entity' convention for clarity and uniqueness.

Output: 13 components using useSWR with inline Supabase fetchers. No more createClient() inside useEffect. Supabase queries benefit from SWR caching, deduplication, and automatic revalidation.
</objective>

<!-- Supabase direct query to SWR conversion pattern (from research) -->
```typescript
// BEFORE:
useEffect(() => {
  const supabase = createClient();
  supabase.from("table").select("*").then(({ data }) => setState(data));
}, []);

// AFTER:
const { data, isLoading } = useSWR(
  'supabase:entity-name',  // unique, stable cache key
  async () => {
    const supabase = createClient();
    const { data, error } = await supabase.from("table").select("*");
    if (error) throw error;
    return data;
  },
  { ...SWR_TIERS.SLOW }
);
```
