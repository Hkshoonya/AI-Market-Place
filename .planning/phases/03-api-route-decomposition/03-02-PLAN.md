Rewrite `src/app/api/marketplace/purchase/route.ts` to be a thin HTTP wrapper that delegates to the handler functions.

The route should keep:
1. `export const dynamic = "force-dynamic"`
2. The Zod purchaseSchema (stays in route — it's HTTP input validation)
3. Rate limiting (IP-based, HTTP concern)
4. Body parsing and Zod validation
5. Auth resolution via resolveAuthUser (HTTP concern — reads from request)
6. Listing fetch (shared between both flows — fetch once, pass to handler)
7. Response formatting (convert PurchaseResult to NextResponse)

The route should delegate to:
```typescript
if (isGuest) {
  const result = await handleGuestCheckout(sb, listing, guest_email, guest_name);
} else {
  const result = await handleAuthenticatedCheckout(sb, listing, auth.userId, auth.authMethod);
}
```

Then convert PurchaseResult to the appropriate NextResponse:
- If `result.success === false` and `result.error`: return error response with appropriate status code
  - "insufficient_balance" -> 402
  - "Authentication required" -> 401
  - "already downloaded" -> 409
  - "Cannot purchase own" -> 400
  - "Email required" -> 400
  - Other errors -> 500
- If `result.success === true`: return 201 with result data

The PurchaseResult should include an `httpStatus` field (add to the interface in purchase-handlers.ts) so the route can directly use it for response status codes, avoiding the need to pattern-match error strings. Each handler sets the appropriate HTTP status.

Route should import from:
- `next/server` (NextRequest, NextResponse)
- `zod` (z)
- `@/lib/rate-limit` (rateLimit, RATE_LIMITS, getClientIp, rateLimitHeaders)
- `@/lib/supabase/admin` (createAdminClient)
- `@/lib/auth/resolve-user` (resolveAuthUser)
- `@/lib/marketplace/purchase-handlers` (handleGuestCheckout, handleAuthenticatedCheckout)

Remove all imports of: wallet, escrow, delivery — those are now internal to purchase-handlers.ts.
