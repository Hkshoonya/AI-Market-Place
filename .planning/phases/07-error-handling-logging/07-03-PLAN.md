For each route file, apply this mechanical migration:

1. **Add imports** at the top:
```typescript
import { handleApiError } from "@/lib/api-error";
```
For cron routes, also add: `import { createTaggedLogger } from "@/lib/logging";` and create `const log = createTaggedLogger("cron/{route-name}");`

2. **Wrap each handler's try/catch** to use handleApiError:
```typescript
// Before:
} catch (err) {
  console.error("[route-name] Error:", err);
  return NextResponse.json({ error: "..." }, { status: 500 });
}
// After:
} catch (err) {
  return handleApiError(err, "api/{route-path}");
}
```

3. **Replace inline console.error/warn** with systemLog or tagged logger calls. For routes that only have 1-2 log calls, use systemLog directly. For cron routes with multiple calls, use createTaggedLogger.

4. **For routes without try/catch** (bare Supabase error checks like bookmarks): Wrap the handler body in a try/catch that calls handleApiError. Keep the existing Supabase error checks (they return 4xx/5xx for expected cases). The outer try/catch captures unexpected throws.

5. **Source naming convention:**
- Admin routes: `"api/admin/{name}"` (e.g., `"api/admin/moderate"`)
- Cron routes: `"cron/{name}"` (e.g., `"cron/compute-scores"`)
- Auth routes: `"api/auth/{name}"`
- Agent routes: `"api/agents/{name}"`

6. **Special cases:**
- `admin/verifications/route.ts`: Replace the 2 console.error calls for notification failures with `systemLog.warn("api/admin/verifications", "Failed to insert notification", { error: ... })` — these are non-critical
- `admin/moderate/route.ts`: Replace 4 console.error calls similarly
- `auth/delete-account/route.ts`: Replace 3 console.error calls with structured logging
- `cron/auctions/route.ts`: Use tagged logger for the 2 console.error calls
- `cron/compute-scores/route.ts`: Use tagged logger for the console.error call
