<!-- Key code the executor needs to understand for auth mocking -->

From src/middleware.ts:
- Middleware calls `supabase.auth.getUser()` on every request
- This makes a POST to `https://<project>.supabase.co/auth/v1/user` with JWT from cookies
- Protected routes: /profile, /sell, /watchlists, /activity, /settings/*, /dashboard/*, /orders/*, /admin/*
- If not authenticated on protected route, redirects to `/login?redirect=<pathname>`

From src/app/(auth)/login/login-form.tsx:
- Client component with `"use client"`
- Uses `supabase.auth.signInWithPassword({ email, password })`
- This POSTs to `**/auth/v1/token?grant_type=password**`
- On success: `router.push(redirectTo)` then `router.refresh()`
- Email input: `aria-label="Email address"`, id="login-email"
- Password input: `aria-label="Password"`, id="login-password"
- Submit button: `button[type="submit"]`, text "Sign In" / "Signing in..."
- Error display: `role="alert"` div

From package.json scripts:
- `"dev": "next dev"` -- webServer command
- `"test": "vitest run"` -- existing unit test script

    2. **Test: 'login with redirect parameter returns to original page'**
       - Same auth interceptions as test 1
       - Navigate to `/login?redirect=/profile`

       - Assert URL becomes `/profile` (or verify redirect was attempted -- middleware may redirect back to login if profile page's server component fails, so assert the form submission succeeded and router.push was called)

    3. **Test: 'invalid credentials show error message'**
       - Intercept `**/auth/v1/token?grant_type=password**` with route.fulfill returning status 400, body: `{ error: 'invalid_grant', error_description: 'Invalid login credentials' }`
       - Navigate to `/login`
       - Fill and submit form
       - Assert error alert visible: `await expect(page.getByRole('alert')).toBeVisible()`

       - For logout: find and click logout button/link in user menu, assert redirect or that "Sign In" appears again

    Important: Register ALL route intercepts BEFORE navigation (before page.goto). The middleware fires on navigation and needs /auth/v1/user to be intercepted already.

    Use Playwright's built-in `test` and `expect` from '@playwright/test'. Use `page.getByLabel()`, `page.getByRole()`, `page.getByText()` for selectors (not CSS selectors) per best practices.
