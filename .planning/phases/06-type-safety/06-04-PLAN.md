The compare-client.tsx has 21 `any` occurrences. These fall into three categories:

**A. Helper functions with `any` parameter types (3 functions):**

```typescript

function getCheapestPrice(model: any): number | null { ... }
```

Add import at top: `import type { ModelWithDetails, BenchmarkScore, ModelPricing } from "@/types/database";`

**B. .map() callbacks with `: any` annotations (~15 occurrences):**

Inside JSX, there are patterns like:
```typescript
.map((p: any) => Number(p.input_price_per_million))
.map((p: any) => Number(p.median_output_tokens_per_second))
.some((m: any) => (m.benchmark_scores ?? []).length > 0)
.filter((m: any) => ...)
.map((m: any, i: number) => ...)
(m.benchmark_scores as any[]).map((bs: any) => ...)
(m.model_pricing as any[])
```

Replace each `: any` with the proper type:
- `(p: any)` in pricing context -> `(p: ModelPricing)`
- `(m: any)` in models context -> `(m: ModelWithDetails)`
- `(bs: any)` in benchmark context -> `(bs: BenchmarkScore)`
- `(m.benchmark_scores as any[])` -> `(m.benchmark_scores ?? [])` (already typed via ModelWithDetails)
- `(m.model_pricing as any[])` -> `(m.model_pricing ?? [])` (already typed via ModelWithDetails)

**C. Check the `models` state variable type:**

If `models` is typed as `any[]`, change it to `ModelWithDetails[]`. This will cascade type inference to all the `.map()` and `.filter()` calls, eliminating most of the explicit annotations automatically.

Remove all `// eslint-disable-next-line @typescript-eslint/no-explicit-any` comments in this file.
