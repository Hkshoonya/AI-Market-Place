<objective>
Create the data integrity verification engine and admin API endpoint that checks end-to-end data flow from adapters through the database, detects empty tables, reports freshness violations, and computes per-source quality scores.

Purpose: This is the core backend for Phase 23 -- without this verification logic, there is no way to prove data actually flows from adapters to the DB, or to detect gaps. The quality score (INTG-04) gives admins a single number per source to assess health at a glance.

Output: A shared lib (`data-integrity.ts`) with pure computation logic + an admin API endpoint (`/api/admin/data-integrity`) returning a structured integrity report.
</objective>

5. **computeQualityScore(params: { completeness: number; freshness: number; trend: number }): number** -- weighted average (completeness: 0.4, freshness: 0.4, trend: 0.2), scaled to 0-100, rounded to integer.

7. **verifyDataIntegrity(supabase: TypedSupabaseClient): Promise<DataIntegrityReport>** -- the main function that:
   a. Fetches all data_sources rows
   b. Fetches pipeline_health rows for freshness data
   c. Fetches recent sync_jobs (last 2 per source, ordered by created_at desc) for trend computation
   d. Counts rows in each expected table using `.select("*", { count: "exact", head: true })`
   e. Computes quality score per source
   f. Builds table coverage map (which tables have data, which are empty, which adapters are responsible)
   g. Builds freshness report (stale sources with timestamps and overdue durations)
   h. Returns the full DataIntegrityReport

For the trend computation: compare the most recent sync_job.records_processed with the second-most-recent for each source. If only one sync exists, trend = 1.0 (no baseline).

Use `formatRelativeTime` from `@/lib/format` for the `overdueBy` field -- or compute a simpler "Xh Ym" or "Xd Yh" string directly (since formatRelativeTime returns "X ago" format which is not ideal here). Create a local `formatDuration(ms: number): string` helper.

Write tests first (RED), then implement (GREEN). Focus tests on the pure computation functions (computeCompleteness, computeFreshness, computeTrend, computeQualityScore) since those have clear I/O. Mock Supabase for the verifyDataIntegrity integration test.
