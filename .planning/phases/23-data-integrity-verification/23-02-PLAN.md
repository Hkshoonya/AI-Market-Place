Add a "Data Integrity" section to the existing admin data-sources page. This section should appear BELOW the existing health summary cards and ABOVE the tier-grouped adapter table. Additions:

**1. New SWR hook at the top of the component:**
```typescript
const { data: integrityData, isLoading: integrityLoading, mutate: mutateIntegrity } = useSWR<DataIntegrityReport>(
  "/api/admin/data-integrity",
  { ...SWR_TIERS.SLOW }
);
```

Define the `DataIntegrityReport`, `SourceQualityScore`, `TableCoverage` interfaces locally in the page's Types section (same pattern as existing `PipelineHealthDetail` etc. -- do NOT import from lib since this is a client component fetching via API, the types are just for local shape).

**2. Quality Score Summary Cards row** (3 cards in a row, matching existing health summary card style):
- **Average Quality**: Large number (0-100) with color coding (green >= 70, amber 40-69, red < 40). Subtitle: "across {N} sources"
- **Stale Sources**: Count of stale sources. Red if > 0, green if 0. Subtitle: "{N} overdue for sync"
- **Empty Tables**: Count of empty tables that should have data. Red if > 0, green if 0. Subtitle: "{N} tables missing data"

**3. "Run Verification" button** next to the section header. On click:
- Call `mutateIntegrity()` to refetch (the API endpoint runs the verification as a side effect of the GET)
- Show a toast: "Verification complete" on success
- Button shows Loader2 spinner while `integrityLoading` is true

**4. Per-source quality score indicators in the existing adapter rows:**
In each adapter row (the ones already rendered in the tier-grouped table), add a quality score badge/pill showing the 0-100 score with color coding:
- Green (>= 70): `bg-gain/10 text-gain`
- Amber (40-69): `bg-amber-400/10 text-amber-400`
- Red (< 40): `bg-loss/10 text-loss`

To find the quality score for each adapter, look up `integrityData?.qualityScores.find(q => q.slug === source.slug)`.

**5. Table Coverage section** below the quality summary cards:
A compact horizontal list of table "pills" showing each tracked table with its row count and empty/populated status. Use the existing Badge component:
- Populated table: green badge with table name and row count
- Empty table: red badge with table name and "0 rows -- {adapter names}" tooltip

**6. Stale Sources detail** (conditional):
If `integrityData?.freshness.staleSources.length > 0`, show a collapsible "Stale Sources" alert panel (amber background) listing each stale source with:
- Source name
- Last sync time (using formatRelativeTime)
- Overdue by (from the API response `overdueBy` field)
- Expected interval

Style everything to match the existing dark theme (bg-card, text-foreground, border-border patterns used throughout the page). Use the same Card/CardContent wrappers. Do NOT add a separate route or page -- this is all inline in the existing data-sources page.

IMPORTANT: The page is already ~700 lines. Keep additions focused and compact. Reuse existing STATUS_CONFIG and HEALTH_CONFIG patterns. The quality score color logic can be a simple helper function at the top:
```typescript

}
```
