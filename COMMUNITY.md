# Community

AI Market Cap is public by design, but it still needs clear channels and working rules.

## Public channels

- GitHub issues: bug reports, feature requests, focused public discussion
- GitHub pull requests: proposed code, docs, design, and infrastructure changes
- [`reports/revenue`](./reports/revenue): public revenue reporting and collaborator distribution record
- [README.md](./README.md), [CONTRIBUTING.md](./CONTRIBUTING.md), [GOVERNANCE.md](./GOVERNANCE.md), and [REVENUE.md](./REVENUE.md): the operating documents for the repo

## Private channel

Use the contact form for:
- security disclosures
- sensitive abuse or conduct reports
- partnership or sponsorship discussions
- brand permission requests

Official contact:
- https://aimarketcap.tech/contact

## How to work here

- open an issue when the change needs alignment first
- keep pull requests narrow and reviewable
- include verification with every meaningful change
- do not bury policy, monetization, or trust-affecting changes inside unrelated PRs

## Collaboration ladder

1. Contributor
Open issues, submit pull requests, improve docs, tests, design, or product behavior.

2. Active collaborator
Contribute repeatedly with merged work, good review participation, and reliable technical judgment.

3. Reviewer
Help enforce quality on public contributions and maintain project standards.

4. Maintainer
Own merge, release, policy, and project-direction responsibility.

## Public accountability

The project should remain inspectable in the following ways:
- revenue reports are committed publicly
- governance rules are written in the repository
- review standards are visible before someone contributes
- brand and license terms are documented clearly

If something affects trust, contributors should expect it to be discussed and documented in public whenever practical.
