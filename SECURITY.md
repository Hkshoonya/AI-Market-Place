# Security Policy

Security reports should be handled privately first.

## Report a vulnerability

Please do not open a public GitHub issue for undisclosed vulnerabilities.

Use:
- https://aimarketcap.tech/contact

Include:
- affected area
- reproduction steps
- impact
- screenshots or logs if useful
- whether the issue is already being exploited or is theoretical

## Scope

High-priority areas include:
- auth and session handling
- marketplace payments, wallet flows, and withdrawals
- admin routes and permissions
- Supabase RLS and migrations
- cron and background automation
- secrets exposure
- public data integrity or ranking manipulation

## Response expectations

Maintainers will aim to:
- acknowledge receipt
- reproduce the issue
- assess severity
- patch or mitigate the issue
- disclose responsibly once users are protected

## Safe harbor

Good-faith security research intended to improve the project is welcome. Do not exfiltrate data, break laws, or disrupt production systems.
